| Dato | Descrizione | Obbligatorietà |

### 2.5 Dati NON Raccolti

Il Bot **NON** raccoglie in alcun modo:
- Dati particolari ai sensi dell'art. 9 GDPR (origine razziale o etnica, opinioni politiche, convinzioni religiose, dati genetici, dati biometrici, dati relativi alla salute, dati relativi alla vita sessuale o all'orientamento sessuale)
- Dati giudiziari ai sensi dell'art. 10 GDPR
- Password o credenziali di autenticazione
- Dati di pagamento o finanziari
- Messaggi privati (DM) non esplicitamente indirizzati al Bot
- Indirizzi IP o dati di geolocalizzazione
- Dati di navigazione o cookie

Ai sensi dell'art. 6(1)(f) del GDPR, il Titolare ha condotto una valutazione di bilanciamento (Legitimate Interest Assessment - LIA) che ha determinato la prevalenza del legittimo interesse del Titolare sui diritti e libertà degli interessati per le seguenti ragioni:
- I dati trattati sono strettamente necessari al funzionamento del servizio
- Le misure di sicurezza implementate riducono significativamente i rischi
- Gli utenti hanno aspettative ragionevoli sul trattamento in base al contesto
- Sono previsti meccanismi efficaci per l'esercizio dei diritti

## SEZIONE 4 - MODALITÀ DEL TRATTAMENTO

### 4.1 Strumenti di Trattamento

Il trattamento dei dati avviene mediante:
- **Database SQLite**: memorizzazione strutturata dei dati con cinque tabelle dedicate (ticket_stats, violazioni_sicurezza, ticket_attivi, blacklist, rate_limit)
- **File system crittografato**: archiviazione dei ticket chiusi con crittografia AES
- **Memoria volatile**: elaborazione temporanea durante l'esecuzione dei comandi

### 4.2 Crittografia dei Dati

Il Bot implementa un sistema di crittografia avanzato per la protezione dei dati sensibili:

| Componente | Algoritmo | Dettagli |
|------------|-----------|----------|
| Derivazione chiave | PBKDF2-HMAC-SHA256 | 100.000 iterazioni |
| Crittografia simmetrica | Fernet (AES-128-CBC + HMAC-SHA256) | Chiave a 256 bit |
| Integrità dati | SHA-256 Checksum | Verifica su ogni archivio |

### 4.3 Sanitizzazione degli Input

Tutti i dati inseriti dagli utenti sono sottoposti a:
- Escape dei caratteri HTML per prevenzione XSS
- Rimozione caratteri di controllo
- Filtraggio pattern potenzialmente pericolosi (javascript:, data:, vbscript:)
- Troncamento a lunghezza massima configurabile

### 4.4 Logging Sicuro

I log di sistema implementano:
- Mascheramento automatico di token e password
- Rotazione file con limite di 10MB per file
- Mantenimento massimo di 20 file di backup
- Encoding UTF-8 per caratteri speciali

## SEZIONE 5 - PERIODO DI CONSERVAZIONE DEI DATI

### 5.1 Tabella dei Periodi di Conservazione

| Categoria Dati | Periodo Conservazione | Criteri di Determinazione |
|----------------|----------------------|---------------------------|
| Ticket attivi | Fino alla chiusura + archiviazione | Necessità operativa |
| Ticket archiviati (attivo) | 14 giorni o 50 ticket max | Configurazione sistema |
| Ticket archiviati (esterno) | Indefinito (compresso) | Finalità storiche |
| Log di interazione | 30 giorni | Rotazione automatica |
| Log di errore | 90 giorni | Necessità debugging |
| Dati rate limiting | 60 secondi | Finalità anti-abuse |
| Blacklist temporanea | Durata specificata | Decisione amministrativa |
| Blacklist permanente | Fino a rimozione manuale | Decisione amministrativa |
| Violazioni sicurezza | 365 giorni | Finalità di audit |
| Feedback e valutazioni | Indefinito | Miglioramento servizio |

### 5.2 Criteri di Cancellazione

I dati vengono cancellati:
- Automaticamente al termine del periodo di conservazione
- Su richiesta dell'interessato (ove applicabile)
- In caso di revoca del consenso (per dati basati su consenso)
- Alla rimozione del Bot dal server Discord

### 5.3 Archiviazione e Compressione

Il sistema prevede tre livelli di archiviazione:
1. **Archivio Attivo**: ticket recenti, accesso immediato
2. **Archivio Esterno**: ticket più vecchi, accesso differito
3. **Archivio Compresso**: backup settimanale in formato ZIP con verifica integrità

## SEZIONE 8 - MISURE DI SICUREZZA

### 8.1 Misure Tecniche

Il Titolare ha implementato le seguenti misure tecniche ai sensi dell'art. 32 GDPR:

| Misura | Implementazione | Finalità |
|--------|-----------------|----------|
| Crittografia at-rest | Fernet (AES-128-CBC) | Protezione dati archiviati |
| Sanitizzazione input | HTML escape, regex filtering | Prevenzione injection |
| Rate limiting | Max 10 comandi/minuto | Prevenzione DoS |
| Blacklist automatica | Dopo 5 violazioni/24h | Protezione da abusi |
| Checksum integrità | SHA-256 | Verifica integrità archivi |
| Logging sicuro | Mascheramento dati sensibili | Protezione credenziali |
| Backup automatici | Ogni 6 ore | Business continuity |

### 8.2 Misure Organizzative

| Misura | Descrizione |
|--------|-------------|
| Principio del minimo privilegio | Accesso ai dati limitato ai soli autorizzati |
| Segregazione ruoli | Distinzione tra admin, staff e moderatori |
| Supervisione diretta | Il Bot opera solo sotto controllo del Titolare |
| Audit trail | Registrazione di tutte le operazioni sensibili |

### 8.3 Gestione delle Violazioni (Data Breach)

In caso di violazione dei dati personali, il Titolare si impegna a:
- Notificare l'Autorità Garante entro 72 ore dalla scoperta (ove richiesto)
- Informare gli interessati senza ingiustificato ritardo (ove necessario)
- Documentare la violazione e le misure adottate
